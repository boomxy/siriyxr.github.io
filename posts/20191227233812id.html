前言

  之前工作室的师兄做了一个基于python人脸识别库的心率监测系统，当时师兄在工作室做技术报告演示的时候就特别感兴趣。这两天研考完了就找师兄要过来研究把玩一下，没想到在环境配置过程中踩坑无数，还差点把我毕设的环境搞崩了，我也真是憨批，明明就有预感拿毕设的虚拟环境来配会出问题，结果就真出问题了。到处查阅资料，倒腾了一下午才把环境恢复过来并最终成功配置运行。

安装包

  有了环境以后下一步就是装包了。先预览一下我们环境最终需要的包。

  其中被我标红的几个是我们接下来需要手动安装的包，可以看到我是使用的pip命令来进行安装的，之前一直觉得conda命令安装要舒服一些，把镜像改成清华镜像以后下载速度也很舒服，还会自动安装相关依赖。但奈何今天要安装的包清华镜像没有资源，使用conda命令安装下载速度贼慢，而且老是下载到一半就卡住断开连接了，反倒是pip要快一些。

  首先要注意一点的是，我新环境装好后pip的版本是9.0.1，而我此时的最新版是19.3.1，就因为这个问题直接导致我之后安装scikit-learn 0.18时失败，所以我们一开始还是先把pip升级到最新版，磨刀不误砍柴工。

  但是更新以后老版仍然存在，既然不影响我是用那就不管了。

运行程序

  该装的包也装完了，但并不意味着我们的程序就能正确运行了，在我运行的时候出现了错误：

  我在这篇博客中找到了问题的解决方案，原来scikit-learn 0.18中代码有问题，我们需要手动修改包中的代码。

解决方法：
  修改lib\site-packages\sklearn\model_selection\_split.py中：
  from scipy.misc import comb为from scipy.special import comb

  修改lib\site-packages\sklearn\metrics\cluster\supervised.py中：
  from scipy.misc import comb为from scipy.special import comb

  至此程序总算可以正常运行了，和我的荣耀手环比对了一下，测量结果基本上偏差不大，师兄果然是牛皮啊！
