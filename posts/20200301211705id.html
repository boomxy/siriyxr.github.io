重邮2016计算机网络（803）真题

一、单选题

1. C

2. D
  工作在物理层的设备是集线器；网桥、交换机工作在数据链路层；路由器工作在网络层。

3. C
  如果采用共享以太网，则每台计算机均分带宽；如果采用交换以太网，则每台计算机带宽与网络总带宽相同。

4. B
  由于这是一个C类地址，默认子网掩码为255.255.255.0，所以网络号为201.100.200.0，主机号为1。

5. B
  DNS即域名解析系统，负责将域名转换为IP地址。

6. D
  HTTP协议会话的四个过程分别为：建立连接、发出请求信息、发出响应信息、释放连接。
【11年单选19题原题】

7. B
  OSI/ISO参考模型中，网络层的主要功能是路由选择、拥塞控制与网络连接；传输层提供可靠的端到端服务，透明的传送报文；数据链路层在通信实体之间传送以帧为单位的数据；表示层提供数据格式转换、数据加密与解密、数据压缩与恢复的功能。
【15年单选20题类似】

8. B
  子网掩码中“1”代表网络部分，由网络号和子网号组成，“0”代表主机部分。

9. B
  以太网主要用于局域网，仅涉及物理层和数据链路层两层，通过MAC地址来区分不同的设备。

10. C
  TCP是面向连接的协议，UDP是面向无连接的协议，但他们都是传输层的协议。

11. D
  该地址为B类地址，前16位为网络号，要划分为9个子网且不是CIDR不能使用全0和全1子网号，则需从主机部分中再划分4为出来。所以子网掩码前20位为1，后面全是0，即225.225.240.0。

12. B
  浏览器和WWW服务器之间传输网页使用的协议是HTTP超文本传输协议。
  IP是网络层协议，FTP是文件传输协议，Telnet是远程登录协议。

13. C
  单工仅能单向传送数据；半双工和双工可双向传输数据，但半双工同一时刻只能一方传输。所以要同一时刻双向传输应使用全双工通信。

14. B
  TCP/IP体系结构分为4层：网络接口层、网际层、传输层、应用层；
  而OSI/IOS体系结构分为7层：物理层、数据链路层、网络层、传输层、会话层、表示层、应用层。

15. B
  服务是指下层为紧邻的上层提供的功能调用，它是“垂直”的。协议是“水平”的。

16. C
  IP地址采用点分十进制表示，每一个数范围仅在0-255内，C选项地址第三位“286”明显错误，不是一个合法地址。

17. A
  OSFP开放最短路径优先协议和RIP路由信息协议都为域内路由协议。域间路由协议为BGP边界网关协议，用于实现不同自治系统间的路由。

18. B
  电子邮件服务器之间和与用户代理之间推送邮件都是使用SMTP协议，而用户代理向电子邮件服务器请求读取邮件使用的是POP3协议。

19. B
  ICMP网际控制报文协议用于控制数据报传送过程中的差错情况，分为差错报告报文和询问报文两种。

20. A
  路由器转发数据包时首先获取目的IP地址，然后查询路由表，选择路由线路，之后再由ARP协议将IP地址转换为MAC地址，从相应端口转发出去。

二、多选题

21. AC
  对等网络中的计算机数量也是有限制的，B错误；对等网络与基于服务器的网络只是两种不同的工作方式，但不是不同的网络类型，D错误。

22. ABC
  网络中的共享资源除了硬件还有数据和软件等，A错误；误码主要是出现在传输过程中，广域网传输距离更远，所以误码率更高且传输速率更慢，而局域网范围小，误码率更低传输速率更快，B错误；资源子网是实现资源共享功能的及其软件的集合，但数据的传输还是要通过通信子网，C错误；单独的一台计算机不能构成计算机网络，要组成一个计算机网络,需具备的三个条件：至少需要2台计算机连接起来、一个共享传输介质或通信子网、一系列实现网络通信协议的软件和硬件，D正确。

23. AC
  MAC地址长6字节，高24位为厂商代码，低24位为厂商自行分配的网卡序列号，B错误；既然是广播帧，则没有具体的某一台主机的MAC地址，广播的MAC地址全为1，所有广播域范围内的主机都要接收，D错误。

24. BCD
  TCP/IP的IP层仅提供无连接服务，传输层提供连接服务和无连接服务，ISO/OSI正好相反，A错误。

25. ABC
  决定传输报文的最佳路由是网络层的功能，D错误。

四、判断题

31. F
  题目所描述的是光纤的特点，相较于光纤、同轴电缆，双绞线恰恰是最差的。

32. T
  分布式与网络操作系统的区别主要在软件的不同。

33. T

34. F
  路由器属于网络层设备，数据链路层设备为网桥、交换机。

35. F
  TCP与UDP都是传输层协议。

36. T

37. T
  PPP协议是在SLIP协议的基础上发展而来的，它既可以在异步线路上传输，又可在同步线路上使用。

38. F
  以太网交换机由于使用了专用的交换结构芯片，因此交换速率较高。从数量级上看，如果局域网交换机的传输时延为几十微秒，那么网桥的传输时延为几百微秒，而路由器的传输时延为几千微秒。

39. T

40. F
  不同的结点不一定具有相同的层次，如路由器仅有三层，交换机仅有两层等；不同结点相同层次可能由于安装的协议不同而具有不同的功能。

41. T

42. F
  OSI/ISO一共有七层。

43. T
  题目说法没错，但实际上相比TCP/IP体系，OSI/ISO反而层数更多，结构复杂，运行效率极低。

44. F
  TCP/IP协议是一个完整的体系，不只是定义了网络接口层。

45. T
  TCP/IP由于得到广泛的应用而成为事实上的国际标准。

六、计算与简答题

51.
物理层要解决的主要问题：

  1. 物理层尽可能屏蔽掉物理设备、传输媒体和通信手段的不同，使上面的数据链路层感觉不到这些差异的存在，而专注于完成本层的协议与服务。
  2. 给其用户（数据链路层）在一条物理的传输媒体上传送和接收比特流（一般为串行按顺序传输等比特流）的能力。因此，物理层应解决物理连接的建立、维护和释放问题。
  3. 在两个相邻系统之间唯一的标识数据电路。

物理层的特点：

  1. 由于在OSI之前，许多规程或协议已经制定出来了，而且在数据通信领域中，这些物理规程已被许多商品化的设备所采用。加之，物理层协议涉及的范围广泛，所以至今没有按OSI的抽象模型制定一套新的物理层协议，而是沿用已存在的物理规程，将物理层确定为描述与传输媒体接口的机械、电器、功能和规程特性。
  2. 由于物理连接方式很多，传输媒体种类也很多，因此，具体的物理协议相当复杂。
    【教材课后习题 2-1】

52.

  1. RIP是应用层协议，只和邻站交换信息，使用UDP无可靠保障，但开销小，可以满足RIP的要求；OSPF是网络层协议，使用可靠的洪泛法，直接使用IP，灵活、开销小；BGP需要交换整个路由表和更新信息，TCP提供可靠交付以减小带宽消耗。
  2. RIP使用不保证可靠交付的UDP，因此必须不断的（周期性）和邻站交换信息才能使路由信息及时得到更新。但BGP使用保证可靠交付的TCP，因此不必这样做。
    【教材课后习题 4-39】

53.
  以太网CSMA/CD的工作流程为：

  1. 先听后发：先侦听信道是否空闲，如果空闲则发送数据，否则继续侦听直到信道空闲立即发送数据。（1坚持）
  2. 边听边发：一边发送一边继续侦听有无其他站点发送的电信号。
  3. 冲突停发：一旦发生冲突立即停止发送数据，取而代之传输一个48比特的拥塞信号，加剧信道的冲突，使其他主机也侦测到冲突的发生。
  4. 随机重发：在发生冲突后采用二进制退避算法等待一个随机时间后再开始侦听信道发送数据。

54.
  数据报丢失重传时，IP数据报的标示字段会有另一个标示符。仅当标示相同的IP数据报片才能组成一个IP数据报。前两个IP数据报片的标识符与后两个IP数据报片的标识符不同，因此不能组装成一个IP数据报。

七、综合题

57.
（1）原来的网络前缀是26，由于要划分4个一样大的子网，且使用的是CIDR，允许使用全0全1子网号，因此需要增加2位。因此每个子网的前缀是28位。

（2）由于每个子网的主机位=32-28=4位，所以共有$2^4=16$个地址，其中14个为主机可用地址。

（3）四个子网的地址分别为：

  1. 136.23.12.0100｜0000=136.23.12.64/28
  2. 136.23.12.0101｜0000=136.23.12.80/28
  3. 136.23.12.0110｜0000=136.23.12.96/28
  4. 136.23.12.0111｜0000=136.23.12.112/28

其中
子网1，分配给主机使用的最小地址为：136.23.12.65；最大地址为：1:6.23.12.78；
子网2，分配给主机使用的最小地址为：136.23.12.81；最大地址为：1:6.23.12.94；
子网3，分配给主机使用的最小地址为：136.23.12.97；最大地址为：1:6.23.12.110；
子网4，分配给主机使用的最小地址为：136.23.12.113；最大地址为：1:6.23.12.126；

58.
（1）HTTP协议在运输层是面向连接的。原因在于，HTTP协议本身虽然是无连接的，但是在运输层使用TCP连接，TCP是面向连接的。

（2）由图可知，选中的数据包使用的GET请求，含义是浏览器采用GET方法向服务器获取资源，也就是说该数据包是从客户端发往服务器。那么，客户端的IP即为113.251.216.102，服务器端的IP地址为119.84.111.126。

（3）HTTP超文本传送协议定义了浏览器（即万维网客户进程）怎样向万维网服务器请求万维网文档，以及服务器怎样把文档传送给浏览器。从层次的角度看，HTTP是面向应用层的协议，它是万维网上能够可靠的交换文件的重要基础。该HTTP执行的操作是向服务器请求资源。
